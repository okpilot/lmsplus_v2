## Durable knowledge

- **CREATE OR REPLACE trace before flagging.** Before flagging a missing pattern (guard, search_path, auth check) on a Postgres function, trace the chain to the LATEST migration definition — the guard may already exist. Grep BOTH `CREATE OR REPLACE FUNCTION <fn>` AND bare `CREATE FUNCTION <fn>` (DROP+recreate form, used when RETURNS TABLE changes): a CREATE-OR-REPLACE-only grep returns a superseded file as "latest" (caught 2026-07-11 — get_quiz_questions cited to harden_strip when diagram_label's DROP+CREATE was 2 migs later). Required pre-flag step per `agent-critic.md`. (Verified on #622 mig 076→077, #629 080→081, #108 void_internal_exam_code.)
- **Migration source of truth: `supabase/migrations/` ONLY.** The old dual-directory invariant is retired — `packages/db/migrations/` was FROZEN 2026-07-11 (historical only, missing 81/222 files; see its README). Never flag a missing packages/db counterpart, and never cite that dir for current SQL.
- **Security §11 (multi-permissive RLS) self-scope is load-bearing.** Per-caller RPCs reading multi-permissive tables must keep the explicit `<owner> = auth.uid()` predicate — RLS ORs the broader policy. Do not suggest removing it. [full → topics/commit-notes.md § row detail 2026-07-11]
- **types.ts nullable-SQL-column convention.** RPC entries may type nullable SQL columns as non-nullable; production query files carry their own local Row type. Not a deviation; SUGGESTION at most. [full → topics/commit-notes.md § row detail 2026-07-11]
- **types.ts stale column after DROP+CREATE migration: ISSUE class when not in staged diff.** When a DROP+CREATE removes a RETURNS TABLE column, types.ts must drop it too — flag ISSUE if absent from the diff (#471). [full → topics/commit-notes.md § row detail 2026-07-11]
- **`rpc`/`authRpc` wrapper contract.** Returns `{ data, error }`, never throws on query errors. `Promise.all([rpc(...), supabase.from(...)])` carries no unhandled-rejection risk. `fetchAllRows` guarantees `data: T[]` (never null) — `?? []` after it is redundant.
- **Doc-only commits: mig comment vs guard line range.** When a citation spans comment+code, the code-only sub-range is more precise (mig 117; clean on #918). [full → topics/commit-notes.md § row detail 2026-07-11]
- **Count/page filter symmetry (pagination).** When a read is split into count + paginated page query, the WHERE filters must be byte-identical — a mismatch fetches wrong/excess rows. Verified clean across #668; keep as a checklist item.
- **Test title impl-detail leakage (code-style §7).** `it(...)` titles must not name internal helpers/types/validator branches; public props/SDK methods/RPC names ARE permitted. Audit inline comments after a title rename. [full → topics/commit-notes.md § row detail 2026-07-11]
- **Dead mock branches in test helpers — ISSUE class, not cosmetic.** Remove `if (table === 'X')` branches for tables the SUT no longer reads so the `Unexpected table` throw fires (PR-A1 dashboard.test.ts). [full → topics/commit-notes.md § row detail 2026-07-11]

### Cached-role pattern in SECURITY DEFINER RPCs

- **Cached role variable prevents NOT NULL abort on delayed soft-delete.** Fetch the actor's role once into a local var at authz time — an inline audit-INSERT subquery returns NULL on mid-txn soft-delete, aborting on NOT NULL. §10 deleted_at filter still required on the capturing SELECT. [full → topics/commit-notes.md § row detail 2026-07-11]
- **NULL-org guard doubles as NULL-role guard when role is fetched on the same SELECT.** After `SELECT u.organization_id, u.role INTO v_admin_org, v_admin_role`, the `IF v_admin_org IS NULL` guard ensures v_admin_role is non-null too. No separate guard needed.
- **ELP grader (`write_oral_section_grade`)** — section-row → session-row lock order with no inverse ordering, so no deadlock; its `auth.uid() IS NOT NULL → RAISE 'forbidden'` guard is defense-in-depth for a service-role-only caller. Do NOT flag it as blocking the grader path. [full → topics/commit-notes.md § row detail 2026-07-11]

### Tooling/config

- **knip `ignoreDependencies` is workspace-scoped.** Use `ignoreDependencies` inside the workspace key (not top level); `ignoreBinaries` + `ignore` live at top level. Verified clean on #325.
- **`@repo/ui` is listed as a dep in `apps/web/package.json` with no `@repo/ui` import** — `packages/ui/src/index.ts` exports `{}` (Phase 5 placeholder). Ignoring it in knip is intentional (forward-declared workspace dep).
- **Broad grep for component names returns false-positive matches** when siblings use same-named primitives from `@base-ui/react` directly (`SelectSeparator` uses `SelectPrimitive.Separator`). Verify import path, not just symbol name.
- **Tailwind v4 `@plugin` directive placement** — after all `@import`, before `@custom-variant`/`@theme`. Verified #325.
