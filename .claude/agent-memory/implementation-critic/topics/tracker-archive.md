- **Trigger-vs-RLS ordering for BEFORE triggers on quiz_sessions (PR-A, 2026-06-04).** For UPDATE: USING is evaluated on the OLD row (row visibility). BEFORE ROW trigger fires next. WITH CHECK is evaluated AFTER the trigger, on the NEW row. If the trigger raises, the transaction aborts before WITH CHECK runs. So a student UPDATE on any frozen column (including student_id/organization_id) surfaces the trigger's RAISE message, NOT a 42501 from WITH CHECK. The `expect(error?.message).toContain('immutable')` assertion in the parametrized #615 tests is definitively correct. Durable rule: when a BEFORE trigger raises on a frozen column, WITH CHECK never runs — do not question trigger-vs-RLS ordering on BEFORE triggers.
